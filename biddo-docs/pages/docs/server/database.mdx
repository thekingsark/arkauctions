

When running the app in development mode, you have the possibility to run it using Docker contains. If you are doing this, a database will automatically be created. 

When it comes with the production mode, I strongly recommend you to use a remote database. Here is a list of free services that you can use:

## Free Cloud PostgreSQL Hosting Services

<br/>
<p>These services provide free-tier PostgreSQL databases with some limitations:</p>
<br/>
<ul>
    <li><strong><a target="_blank" href="https://neon.tech">Neon.tech</a></strong> – Serverless PostgreSQL, 1 GB free storage.</li>
    <br/>
    <li><strong><a target="_blank" href="https://supabase.com">Supabase</a></strong> – Managed PostgreSQL with APIs, 500 MB free.</li>
    <br/>
    <li><strong><a target="_blank" href="https://render.com">Render</a></strong> – Free PostgreSQL (100 MB storage, sleeps on inactivity).</li>
    <br/>
    <li><strong><a target="_blank" href="https://www.elephantsql.com">ElephantSQL</a></strong> – Free Tiny Turtle plan (20 MB storage).</li>
    <br/>
    <li><strong><a target="_blank" href="https://aiven.io">Aiven</a></strong> – Free trial with 5 GB PostgreSQL database.</li>
    <br/>
</ul>
